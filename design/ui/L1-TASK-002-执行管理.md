| 属性 | 值 |
|------|-----|

| 时间范围 | `a-range-picker` | 本地状态 | 按执行时间过滤 |

| 执行记录表格 | `a-table` | API: GET executions | — |

## 交互行为

| 用户操作 | 触发事件 | 页面响应 |
|----------|----------|----------|
| 点击「触发执行」 | 打开 Modal | 显示执行配置表单 |
| 选择用例范围并提交 | POST /api/.../executions | 关闭弹窗，新执行记录出现在表格顶部，状态为「排队中」 |
| 选择状态筛选 | GET executions?status=x | 表格按状态过滤 |
| 选择时间范围 | GET executions?startTime=x&endTime=y | 表格按时间过滤 |
| 点击行「查看」 | 路由跳转 | 进入执行详情页 `/projects/:id/executions/:eid` |
| 点击行「重新执行」 | 弹出确认框 | 确认后 POST 重新触发，新记录出现 |
| 点击行「停止」（运行中） | 弹出确认框 | 确认后 POST cancel，状态变为「已取消」 |
| 执行进行中 | WebSocket/轮询 | 状态实时更新（每 5s 轮询或 WS 推送） |
| 在详情页点击失败用例行 | 展开行 | 显示错误信息和截图（如有） |
| 切换分页 | GET executions?page=x | 表格更新 |

## 状态覆盖

| 状态 | 表现 |
|------|------|
| 加载中 | 表格显示 `loading` 旋转指示器 |
| 空数据 | `a-empty`，描述「暂无执行记录，点击上方按钮触发执行」 |
| 错误 | `a-alert type="error"` + 重试按钮 |
| 正常 | 表格正常展示执行记录 |

## 表单字段（触发执行）

| 字段 | 类型 | 组件 | 校验规则 | 说明 |
|------|------|------|----------|------|
| 执行名称 | string | `a-input` | 必填，自动生成默认值 | 格式：执行-YYYYMMDD-HHmm |
| 执行范围 | enum | `a-radio-group` | 必填 | 全部用例/按功能点/按优先级/自定义选择 |
| 功能点选择 | array | `a-tree-select` multiple | 范围=按功能点时必填 | 多选功能点 |
| 优先级选择 | array | `a-checkbox-group` | 范围=按优先级时必填 | P0/P1/P2/P3 |
| 自定义用例 | array | `a-transfer` | 范围=自定义时必填 | 穿梭框选择具体用例 |
| 执行环境 | enum | `a-select` | 必填 | 本地/CI/远程 |

## 调用的后端 API

| API | 方法 | 触发时机 | 用途 |
|-----|------|----------|------|
| `/api/projects/:id/executions` | GET | 页面加载、筛选、翻页 | 获取执行记录列表 |
| `/api/projects/:id/executions` | POST | 提交触发执行表单 | 创建新执行任务 |
| `/api/projects/:id/executions/:eid` | GET | 进入详情页 | 获取执行详情 |
| `/api/projects/:id/executions/:eid/results` | GET | 详情页加载 | 获取用例执行结果列表 |
| `/api/projects/:id/executions/:eid/cancel` | POST | 点击停止 | 取消执行 |
| `/api/projects/:id/executions/:eid/retry` | POST | 点击重新执行 | 重新触发执行 |
