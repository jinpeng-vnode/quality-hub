# L1-TASK-002 报告看板页面 UI 规格

## 页面信息

| 属性 | 值 |
|------|-----|
| 路由 | `/projects/:projectId/report` |
| 文件路径 | `src/views/ReportView.vue` |
| 权限 | 登录用户（V1 无认证） |
| 导航位置 | 左侧菜单第 5 项，图标 `DashboardOutlined` |

## 页面布局

## 组件列表

| 组件 | Ant Design 组件 | 数据来源 | 说明 |
|------|-----------------|----------|------|
| 面包屑 | `a-breadcrumb` | 路由参数 + store | — |
| 时间范围选择 | `a-select` + `a-range-picker` | 本地状态 | 预设：最近7天/30天/自定义 |
| 统计卡片 | `a-statistic` 包裹在 `a-card` 中 | API: GET /api/reports/dashboard | 4 个指标卡片 |
| 通过率趋势图 | `vue-echarts` 折线图 | API: GET /api/reports/trend | X 轴日期，Y 轴 passRate |
| 覆盖状态饼图 | `vue-echarts` 饼图 | API: GET /api/reports/dashboard | pending/partial/covered 分布 |
| 最近执行表格 | `a-table` | API: GET /api/runs?projectId=x（取前 5 条） | 简化表格 |
| 状态标签 | `a-tag` | 行数据 | 同执行管理页面配色 |

## 交互行为

| 用户操作 | 触发事件 | 页面响应 |
|----------|----------|----------|
| 切换时间范围预设 | GET /api/reports/trend?days=x | 趋势图刷新 |
| 选择自定义日期范围 | GET /api/reports/trend?days=x | 趋势图刷新 |
| 鼠标悬停图表数据点 | — | 显示 tooltip 详细数值 |
| 点击饼图某个扇区 | 路由跳转 | 跳转到功能点页面并按该状态筛选 |
| 点击最近执行表格某行 | 路由跳转 | 跳转到该执行的详情页 |
| 页面加载 | 并行请求 dashboard + trend + runs | 各区域独立加载 |

## 状态覆盖

| 状态 | 表现 |
|------|------|
| 加载中 | 统计卡片显示 `a-skeleton`，图表区域显示 `a-spin` |
| 空数据 | 统计卡片显示 0/—，图表显示 `a-empty` 占位「暂无数据」 |
| 错误 | 各区域独立错误处理，`a-alert type="error"` + 重试按钮 |
| 正常 | 统计卡片数值 + 图表正常渲染 |

## 调用的后端 API

| API | 方法 | 触发时机 | 用途 |
|-----|------|----------|------|
| `/api/reports/dashboard?projectId={id}` | GET | 页面加载 | 获取覆盖率+通过率统计 |
| `/api/reports/trend?projectId={id}&days={n}` | GET | 页面加载、切换时间范围 | 获取通过率趋势数据 |
| `/api/runs?projectId={id}` | GET | 页面加载 | 获取最近执行记录（前端取前 5 条） |
