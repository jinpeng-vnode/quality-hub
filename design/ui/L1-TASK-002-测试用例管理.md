# L1-TASK-002 测试用例管理页面 UI 规格

## 页面信息

| 属性 | 值 |
|------|-----|
| 路由 | `/projects/:projectId/testcases` |
| 文件路径 | `src/views/testcases/TestCaseList.vue` |
| 权限 | 登录用户，需属于当前项目 |
| 导航位置 | 左侧菜单第 3 项，图标 `ExperimentOutlined` |

## 组件列表

| 组件 | Ant Design 组件 | 数据来源 | 说明 |
|------|-----------------|----------|------|
| 面包屑 | `a-breadcrumb` | 路由参数 + store | — |
| 新增按钮 | `a-button type="primary"` | — | 打开新增用例弹窗 |
| 批量操作下拉 | `a-dropdown` + `a-menu` | — | 批量删除、批量修改状态 |
| 功能点筛选 | `a-tree-select` | API: GET features | 按功能点过滤 |
| 状态筛选 | `a-select` | 本地枚举 | 草稿/待审核/已通过/已废弃 |
| 搜索框 | `a-input-search` | 本地状态 | 按用例标题搜索 |
| 左侧功能树 | `a-tree` | API: GET features（树形） | 点击节点过滤右侧表格 |
| 用例表格 | `a-table` | API: GET testcases | 支持行选择（checkbox） |
| 优先级标签 | `a-tag` | 行数据 | P0 红/P1 橙/P2 蓝/P3 灰 |
| 状态标签 | `a-tag` | 行数据 | 不同状态不同颜色 |
| 关联标识 | `a-tooltip` + 图标 | 行数据 | 显示是否关联 Midscene 脚本 |
| 详情抽屉 | `a-drawer` | API: GET testcases/:id | 宽度 720px |
| 抽屉 Tab | `a-tabs` | — | 基本信息/测试步骤/Midscene 脚本/执行历史 |
| 步骤编辑器 | `a-table`（可编辑行） | 用例详情 | 支持拖拽排序 |
| 分页器 | `a-pagination`（表格内置） | API 返回 total | 默认 pageSize=20 |

## 交互行为

| 用户操作 | 触发事件 | 页面响应 |
|----------|----------|----------|
| 点击「新增用例」 | 打开 Modal | 显示新增用例表单 |
| 填写表单并提交 | POST /api/.../testcases | 关闭弹窗，刷新表格 |
| 点击左侧功能树节点 | GET testcases?featureId=x | 右侧表格过滤为该功能点下的用例 |
| 选择状态/功能点筛选 | GET testcases?status=x&featureId=y | 表格按条件过滤 |
| 输入搜索关键词 | GET testcases?keyword=xxx | 表格过滤，防抖 300ms |
| 点击用例行 | 打开 Drawer | 右侧滑出详情抽屉 |
| 在详情抽屉编辑步骤 | 本地状态更新 | 可增删改步骤，拖拽排序 |
| 点击详情抽屉「保存」 | PUT /api/.../testcases/:id | 保存修改，成功提示 |
| 点击「关联 Midscene」 | 打开子 Modal | 输入/编辑 Midscene 脚本路径和配置 |
| 勾选多行 + 批量删除 | 弹出确认框 | 确认后批量 DELETE，刷新表格 |
| 勾选多行 + 批量改状态 | 弹出状态选择 | 确认后批量 PATCH，刷新表格 |
| 点击行「删除」 | 弹出确认框 | 确认后 DELETE，刷新表格 |
| 切换分页 | GET testcases?page=x | 表格更新 |

## 状态覆盖

| 状态 | 表现 |
|------|------|
| 加载中 | 左侧树 `a-spin`，右侧表格 `loading` 属性 |
| 空数据 | 表格内嵌 `a-empty`，描述「暂无测试用例」；左侧树为空时显示「请先创建功能点」 |
| 错误 | `a-alert type="error"` 显示在表格上方 + 重试按钮 |
| 正常 | 左侧树 + 右侧表格正常展示 |

## 表单字段（新增/编辑用例）

| 字段 | 类型 | 组件 | 校验规则 | 说明 |
|------|------|------|----------|------|
| 用例标题 | string | `a-input` | 必填，2-200 字符 | — |
| 关联功能点 | string | `a-tree-select` | 必填 | 从功能点树选择 |
| 优先级 | enum | `a-select` | 必填 | P0/P1/P2/P3 |
| 前置条件 | string | `a-textarea` | 可选 | 执行用例前的准备条件 |
| 状态 | enum | `a-select` | 必填 | 草稿/待审核/已通过/已废弃 |
| 测试步骤 | array | 可编辑表格 | 至少 1 步 | 每步含：步骤描述、预期结果 |
| Midscene 脚本路径 | string | `a-input` | 可选 | 关联的 E2E 脚本文件路径 |
| Midscene 配置 | json | `a-textarea` | 可选，JSON 格式 | 脚本运行参数 |

## 测试步骤子表格字段

| 字段 | 类型 | 组件 | 校验规则 | 说明 |
|------|------|------|----------|------|
| 步骤序号 | number | 自动生成 | — | 支持拖拽调整顺序 |
| 操作描述 | string | `a-input` | 必填 | 用户执行的操作 |
| 预期结果 | string | `a-input` | 必填 | 期望的系统响应 |

## 调用的后端 API

| API | 方法 | 触发时机 | 用途 |
|-----|------|----------|------|
| `/api/projects/:id/features` | GET | 页面加载 | 获取功能点树（左侧导航） |
| `/api/projects/:id/testcases` | GET | 页面加载、筛选、翻页 | 获取用例列表 |
| `/api/projects/:id/testcases` | POST | 提交新增表单 | 创建用例 |
| `/api/projects/:id/testcases/:tid` | GET | 点击用例行 | 获取用例详情（含步骤） |
| `/api/projects/:id/testcases/:tid` | PUT | 详情抽屉保存 | 更新用例 |
| `/api/projects/:id/testcases/:tid` | DELETE | 确认删除 | 删除用例 |
| `/api/projects/:id/testcases/batch` | PATCH | 批量操作 | 批量修改状态 |
| `/api/projects/:id/testcases/batch` | DELETE | 批量删除 | 批量删除用例 |
