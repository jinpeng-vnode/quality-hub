# BUG-029: 级联删除有关联数据时返回500

## 基本信息
- Issue: #29
- 优先级: P1
- 发现时间: 2026-05-26 14:50 UTC
- 模块: 项目管理 / 功能点管理
- 环境: https://quality-hub.todonot.com

## Bug 描述

删除有关联数据的功能点或项目时，API 返回 500 Internal Server Error，数据未被删除。

## 复现步骤

### 场景1：删除有关联用例的功能点
```bash
# 1. 创建功能点并在其下创建用例
curl -X POST "https://quality-hub.todonot.com/api/features" \
  -H "Content-Type: application/json" \
  -d '{"projectId": "<project_id>", "title": "测试功能点"}'

curl -X POST "https://quality-hub.todonot.com/api/cases" \
  -H "Content-Type: application/json" \
  -d '{"featureId": "<feature_id>", "title": "测试用例"}'

# 2. 删除功能点
curl -X DELETE "https://quality-hub.todonot.com/api/features/<feature_id>"
# 返回: 500 {"code":"INTERNAL_ERROR","message":"服务器内部错误","detail":null}
```

### 场景2：删除有关联数据的项目
```bash
# 项目下有功能点、用例、执行记录时
curl -X DELETE "https://quality-hub.todonot.com/api/projects/<project_id>"
# 返回: 500 {"code":"INTERNAL_ERROR","message":"服务器内部错误","detail":null}
```

## 预期行为

- 删除功能点时应级联删除其下所有用例，返回 HTTP 200
- 删除项目时应级联删除其下所有功能点、用例、执行记录，返回 HTTP 200

## 实际行为

- 返回 HTTP 500 `{"code":"INTERNAL_ERROR","message":"服务器内部错误","detail":null}`
- 数据未被删除

## 对比

- 删除无关联数据的功能点/项目：正常返回 200 ✅
- 删除有关联数据的功能点/项目：返回 500 ❌

## 可能原因

数据库外键约束未配置级联删除（CASCADE），或删除逻辑未先删除子记录。

## 影响范围

- 用户无法删除有数据的功能点和项目
- 数据清理功能不可用
