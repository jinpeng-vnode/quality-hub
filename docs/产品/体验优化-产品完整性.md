## 二、待优化问题清单

### P1 — 影响基本可用性

#### 1. 文案国际化不一致
- **现象**：新建项目/功能点/编辑弹窗（Modal）按钮为英文 `Cancel` / `OK`；删除确认气泡也是 `Cancel` / `OK`；但新建用例抽屉（Drawer）按钮为中文「取 消」「保 存」
- **影响**：中文产品出现英文按钮，用户困惑；且同一产品内两种风格并存
- **建议**：统一为中文「取消」「确定」（Modal）和「取消」「保存」（Drawer），通过 Ant Design Vue 的 `locale` 配置全局设置中文

#### 2. 路由直接访问回退问题
- **现象**：直接在浏览器输入 `/features`、`/testcases`、`/test-cases` 等路由，页面回退到项目管理空页面（只有标题，无内容）
- **影响**：用户无法通过 URL 分享或收藏具体页面；刷新页面可能丢失上下文
- **建议**：
  - 无项目上下文时，重定向到 `/projects` 并提示"请先选择项目"
  - 或在 URL 中始终保留项目 ID（当前已实现 `/projects/{id}/features` 格式，但需处理无 ID 的兜底）

#### 3. 无 404 页面
- **现象**：访问不存在的路由（如 `/nonexistent-page`）显示空白的项目管理页面，无任何错误提示
- **影响**：用户无法判断是页面加载失败还是路由错误
- **建议**：添加 404 页面，包含"页面不存在"提示和返回首页链接

---

### P2 — 影响专业感和一致性

#### 4. 页面 title 未动态设置
- **现象**：所有页面的浏览器标签标题始终为 `frontend`
- **影响**：多标签页时无法区分页面；SEO 不友好
- **建议**：根据当前页面动态设置 title，格式如 `功能点管理 - 电商平台 - QHub`

#### 5. 执行详情时间格式未优化
- **现象**：执行详情抽屉中"开始时间"显示原始 ISO 格式 `2026-05-27T00:37:20.507129+00:00`
- **影响**：信息密度高，可读性差
- **建议**：格式化为 `2026-05-27 00:37:20`，或使用相对时间 + tooltip 显示完整时间

#### 6. 执行详情用例 ID 未截断
- **现象**：执行详情抽屉中"用例结果"表格的用例 ID 列显示完整 UUID `fd068c70-03ef-44f1-97bb-88cdf833919c`，导致行高异常（165px）
- **影响**：表格布局严重变形，信息密度极低
- **建议**：与其他表格一致，截断为前 8 位，hover 或点击显示完整 ID

#### 7. 侧边栏无当前项目名称
- **现象**：进入项目后，侧边栏只显示 `QHub` logo 和导航菜单，无法看到当前所在项目
- **影响**：多项目切换时用户无法确认当前上下文
- **建议**：在 logo 下方或导航菜单上方显示当前项目名称，可点击返回项目列表

#### 8. 无面包屑导航
- **现象**：进入项目子页面后，无面包屑显示层级关系（如 `项目管理 > 电商平台 > 功能点管理`）
- **影响**：用户无法快速感知当前位置和层级
- **建议**：在页面标题上方添加面包屑，至少显示 `项目名 / 当前模块`

#### 9. 项目卡片无编辑/删除入口
- **现象**：项目管理页面的卡片只能点击进入，无法编辑项目信息或删除项目
- **影响**：API 支持项目的 PUT/DELETE，但前端无操作入口
- **建议**：卡片右上角添加更多操作菜单（`...`），包含"编辑"和"删除"选项

#### 10. 触发执行按钮交互不符合预期
- **现象**：按钮显示为 `触发执行 ▼`（带下拉箭头），但点击后直接触发了执行，没有展开下拉菜单
- **影响**：下拉箭头暗示有多个选项（如选择执行范围），但实际直接执行，违反用户预期
- **建议**：
  - 方案 A：去掉下拉箭头，改为普通按钮直接触发全量执行
  - 方案 B：保留下拉，展开后提供"全量执行"和"选择用例执行"两个选项
