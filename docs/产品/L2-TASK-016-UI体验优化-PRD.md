# L2-TASK-016 UI 体验优化 产品需求文档

## 1. 需求背景

### 来源
老板反馈 Quality Hub 功能虽已全部通过测试（47 用例 100% 通过），但 UI 效果不理想。

### 现状
- 页面总宽仅 780px，内容区 580px，未充分利用屏幕空间
- 表格 ID 列显示完整 UUID（36 字符），占据大量水平空间
- 页面标题被裁切（y 坐标为负值）
- 操作按钮拥挤，文字换行
- 缺乏 UI 设计规范，无 design/ui/ 文档

### 痛点
1. 布局紧凑导致信息密度过高，可读性差
2. UUID 全量展示无实际意义，浪费空间
3. 表头行高过大（77px），数据行高不一致
4. 百分比数值显示格式异常（小数点与数字分离）
5. 整体缺乏视觉层次和呼吸感

## 2. 目标用户与场景

- **用户**：QA 工程师、项目经理、开发人员
- **场景**：日常使用 Quality Hub 管理项目质量数据，需要快速浏览、筛选、操作
- **核心诉求**：信息清晰、操作高效、视觉舒适

## 3. 功能描述

### 3.1 全局布局优化

| 问题 | 现状 | 优化方向 |
|------|------|----------|
| 页面宽度受限 | 总宽 780px，内容区仅 580px | 内容区自适应，最小宽度 960px，最大宽度 1440px |
| 页面标题裁切 | heading y=-5，被上边界裁切 | 标题区域增加顶部 padding，确保完整显示 |
| 侧边栏比例 | 固定 200px | 保持 200px 但内容区应占满剩余空间 |

### 3.2 项目管理页（/projects）

| 问题 | 现状 | 优化方向 |
|------|------|----------|
| 卡片宽度固定 | 每行 2 个，宽 242px | 响应式网格，根据屏幕宽度自动调整列数（3-4 列） |
| 卡片信息层次不清 | 标题、描述、时间平铺 | 标题加粗突出，描述灰色次要，时间用相对时间 |
| 无项目状态标识 | 纯文字卡片 | 增加项目活跃度指示（最近执行时间） |
| "新建项目"按钮位置 | 左上角小按钮 | 保持位置但增加视觉权重 |

### 3.3 功能点管理页

| 问题 | 现状 | 优化方向 |
|------|------|----------|
| ID 列显示完整 UUID | `97f82546-61e9-4ff1-96d8-54a9b5d97cc4`，占 136px | 隐藏 ID 列或截断显示前 8 位，hover 显示完整 |
| 表头行高过大 | 77px | 统一为 48px |
| 操作列拥挤 | "编辑""删除"文字紧贴 | 改为图标按钮或增加间距 |
| 筛选器与新建按钮布局 | 筛选器和按钮在同一行但间距不合理 | 筛选器左对齐，新建按钮右对齐 |
| 数据行高不一致 | 行高 121px（因 UUID 换行） | 统一行高 56px |

### 3.4 测试用例管理页

| 问题 | 现状 | 优化方向 |
|------|------|----------|
| ID 列完整 UUID | 同上 | 同上处理 |
| 操作列文字换行 | "编辑""删除"垂直排列，box 高 44px | 改为图标按钮，水平排列 |
| 优先级/类型显示 | 英文原始值 "medium"/"manual" | 中文化或使用彩色标签 |
| 筛选器过多 | 功能点选择 + 类型筛选 + 新建按钮挤在一行 | 合理分组，筛选器左侧，操作按钮右侧 |

### 3.5 执行管理页

| 问题 | 现状 | 优化方向 |
|------|------|----------|
| ID 列完整 UUID | 同上 | 同上处理 |
| 表头行高 77px | 过高 | 统一 48px |
| "查看详情"文字按钮 | 无明确视觉引导 | 改为链接样式或图标 |
| 结果列格式 | "1/1 通过" | 增加颜色标识（绿色通过/红色失败） |
| 创建时间格式 | "2026-05-26 15:11:29" | 使用相对时间或简化格式 |

### 3.6 报告看板页

| 问题 | 现状 | 优化方向 |
|------|------|----------|
| 百分比格式异常 | "0" + ".0" + "%" 分成 3 个元素 | 合并为单一文本 "0.0%" |
| 统计卡片间距 | 4 个卡片等宽 113px 平铺 | 增加卡片间距，添加背景色区分 |
| 图表区域 | 通过率趋势 + 覆盖状态并排 | 确保图表有足够宽度，数据为空时显示空状态 |
| 时间筛选器 | "最近 7 天" 下拉 | 位置合理，保持不变 |

## 4. 验收标准

### 全局
- [ ] 页面内容区宽度自适应，在 1280px 屏幕下无水平滚动条
- [ ] 所有页面标题完整显示，无裁切
- [ ] 侧边栏与内容区比例协调

### 表格通用
- [ ] ID 列不显示完整 UUID，改为截断前 8 位 + tooltip 或直接隐藏
- [ ] 表头行高统一为 48px
- [ ] 数据行高统一为 56px
- [ ] 操作列按钮使用图标，水平排列，间距 ≥ 8px

### 项目管理页
- [ ] 卡片网格响应式，≥1280px 时每行至少 3 个
- [ ] 卡片信息层次清晰（标题 > 描述 > 元信息）

### 功能点/测试用例/执行管理页
- [ ] 筛选器左对齐，操作按钮右对齐
- [ ] 优先级/类型/状态使用彩色标签
- [ ] 执行结果使用颜色区分通过/失败

### 报告看板页
- [ ] 百分比数值正确渲染为单一文本
- [ ] 统计卡片有视觉区分（背景色或边框）
- [ ] 图表在数据为空时显示友好的空状态

## 5. 优先级与里程碑

**整体优先级：P2（设计/体验问题，规划修复）**

| 子项 | 优先级 | 说明 |
|------|--------|------|
| ID 列截断/隐藏 | P1 | 影响所有表格页可读性 |
| 页面标题裁切修复 | P1 | 明显视觉缺陷 |
| 表格行高统一 | P1 | 影响信息密度 |
| 操作按钮图标化 | P2 | 改善操作体验 |
| 百分比格式修复 | P2 | 数据展示准确性 |
| 内容区自适应宽度 | P2 | 提升空间利用率 |
| 卡片响应式布局 | P2 | 项目管理页体验 |
| 彩色标签/状态色 | P3 | 视觉增强 |
| 相对时间显示 | P3 | 锦上添花 |

## 7. 非功能性要求

- **性能**：优化不应增加首屏加载时间
- **兼容性**：支持 Chrome/Edge/Firefox 最新版，最小分辨率 1280×720
- **可维护性**：建立统一的 UI 设计规范文档（design/ui/），后续开发参照执行
- **无障碍**：表格保持语义化标签，图标按钮需有 aria-label

## 8. 风险与备选方案

| 风险 | 影响 | 备选方案 |
|------|------|----------|
| 隐藏 ID 列可能影响调试 | 开发者需要 ID 定位问题 | 保留 ID 列但截断，点击可复制完整值 |
| 响应式改动范围大 | 可能引入新的布局问题 | 分阶段实施，先修复明显缺陷再做响应式 |
| 无 UI 设计师资源 | 优化方向可能不统一 | 先制定基础设计规范（间距、颜色、字号），再逐页实施 |
