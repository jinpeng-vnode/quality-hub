## 失败详情

### BUG-1: 项目名称唯一性校验缺失（TC-012）
- **优先级**: P1
- **现象**: 使用相同名称创建项目时返回 200 并成功创建，未返回 409 错误
- **预期**: PRD 明确要求"项目名称唯一"，重复时应返回错误
- **影响**: Issue #5 验收标准未满足

### BUG-2: 功能点名称唯一性校验缺失（TC-022）
- **优先级**: P1
- **现象**: 同一项目下使用相同名称创建功能点时返回 200 并成功创建
- **预期**: PRD 要求"功能点名称在项目内唯一"，重复时应返回错误
- **影响**: Issue #6 验收标准未满足

### BUG-3: 用例标题唯一性校验缺失（TC-033）
- **优先级**: P1
- **现象**: 同一功能点下使用相同标题创建用例时返回 200 并成功创建
- **预期**: PRD 要求"用例标题在功能点内唯一"，重复时应返回错误
- **影响**: Issue #7 验收标准未满足

### BUG-4: 批量创建用例端点缺失（TC-034/035）
- **优先级**: P1
- **现象**: `POST /api/cases/batch` 返回 405 Method Not Allowed，端点不存在
- **预期**: PRD 要求"API 支持批量创建（单次最多 50 条）"
- **影响**: Issue #7 验收标准未满足

### BUG-5: 全局看板端点缺失（TC-053）
- **优先级**: P2
- **现象**: `GET /api/dashboard/overview` 返回 404，`GET /api/reports/dashboard` 不传 projectId 返回 422
- **预期**: PRD 要求"全局看板（所有项目汇总）"
- **影响**: Issue #9 验收标准未满足
